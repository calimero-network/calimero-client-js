Architecture

Five API surfaces, one shared HTTP client — accessed through the top-level apiClient singleton

Package Structure

src/api/
Five typed API surfaces: NodeApi, AuthApi, AdminApi, BlobApi, and RpcClient. All share a single AxiosHttpClient instance that handles JWT attachment and token refresh.
src/api/dataSource/
Concrete implementations of each API interface. Each class extends BaseApiDataSource which provides URL construction helpers.
src/rpc/
JsonRpcClient — sends JSON-RPC 2.0 requests to the node's /jsonrpc endpoint. Handles query and mutate operations on WASM applications.
src/subscriptions/
WsSubscriptionsClient and SseSubscriptionsClient — establish persistent connections and deliver NodeEvent callbacks to the application.
src/login/ + src/setup/
React components: ClientLogin and SetupModal for the guided auth flow. Token storage and refresh handled automatically.
src/storage/
localStorage-backed helpers for node URL, application ID, access token, and refresh token. Used by all API layers.
src/experimental/
Low-level SSE/WebSocket primitives and CalimeroProvider React context. APIs here may change between releases.
src/types/
TypeScript interfaces: RpcQueryParams, NodeEvent, ApiResponse<T>, error types.

RpcClient Design

RpcClient wraps the node's JSON-RPC-over-HTTP endpoint. Every call is either a query (read-only, no state change) or a mutate (writes state):

// Query — read state from the WASM application
const result = await rpcClient.query<{ value: string }>({
  contextId: 'ctx_...',
  method: 'get_value',
  argsJson: JSON.stringify({ key: 'hello' }),
});

// Mutate — write state to the WASM application
const txHash = await rpcClient.mutate({
  contextId: 'ctx_...',
  method: 'set_value',
  argsJson: JSON.stringify({ key: 'hello', value: 'world' }),
});

SubscriptionsClient Design

Supports both WebSocket and Server-Sent Events (SSE). The transport is selected at construction time:

import { SubscriptionsClient, SubscriptionType } from '@calimero-network/calimero-client';

const sub = new SubscriptionsClient(nodeUrl, applicationId, {
  type: SubscriptionType.WebSocket, // or .SSE
});

sub.subscribe((event) => {
  console.log(event);
});

// Clean up:
sub.unsubscribe();

Authentication

Both clients require authentication. The SDK provides login helpers that store tokens in localStorage:

import { login, getAccessToken } from '@calimero-network/calimero-client';

// Login and store tokens
await login(nodeUrl, { username: 'admin', password: '...' });

// Clients automatically read the stored token
const rpc = new RpcClient(nodeUrl, applicationId);
// Bearer header is attached automatically
